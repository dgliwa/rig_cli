## Milestone Summary

**Key Decisions:**

- Named catalog constants (`MOOD_MKII_WET_BYPASS_CC`, `MOOD_MKII_LOOP_BYPASS_CC`) at module level — tests import the constants; YAML remains the runtime source of truth; catalog is authoritative for tests and documentation
- `ActionStatus.VERIFY` used for all device types when preset already matches state — unified status regardless of device type (analog/digital/controller)
- MC6 SysEx `save=True` default for all three write functions — aligns `clear_preset_messages` with sibling functions and MC6 web UI behavior; eliminates power-cycle regression
- `ParamDiff(name, before, after)` Pydantic model on `DeviceAction.param_diff: list[ParamDiff] = []` — zero-cost for HX Stomp and unchanged devices; serializes via Pydantic automatically
- `device_filter: str | None = None` on `apply_plan()` — simpler than a separate engine function; gate on `action.device != device_filter` skips non-matching devices; controller phase gated with `and not device_filter`
- Phase 33.1 inserted to execute Phase 33 work — original Phase 33 slot reserved in roadmap; 33.1 is the executed phase with its own planning artifacts

**Post-Close Blocker Fix (commit a098afb):**

- MC6 controller `apply()` return value was discarded; `state_modified` only set when controller ID already in `state.devices` (always False on cold start)
- Fixed by capturing `controller_result = controller.apply(controller_ctx)` and setting `state_modified = True` when `controller_result.status == "confirmed"`

**Issues Resolved:**

- 2 test failures at milestone start (wet_bypass/loop_bypass CC swap) — resolved Phase 29
- `state.json` not tracking per-device `last_preset` — resolved Phase 30
- MC6 clear SysEx sent `save=False` (RAM-only) instead of `save=True` (flash) — resolved Phase 31
- `rig plan` showing "changed" with no parameter detail — resolved Phase 32
- Analog apply re-prompting even when state matched — resolved Phase 33.1
- MC6 controller-only apply not persisting to state.json — resolved post-close

**Technical Debt Incurred:**

- Pre-existing TODOs in `plan.py:80` ("scene plans should have their own logging output class") and `compute.py:95` ("not a fan of defining functions INSIDE functions") — present before v1.6, unchanged
- CBA validation skips already-saved presets (state-gating) — carried from v1.3 deferred items, still acknowledged
