# Milestone v1.1: Package Extraction & Plugin Isolation

**Status:** ✅ SHIPPED 2026-06-07
**Phases:** 6-8
**Total Plans:** 8

## Overview

Extracted each device system into independently installable pip packages with Setuptools Entry Points for plugin discovery. Device-level MIDI connection management moved to each device plugin. Core (`rig`) has zero hard dependencies on any device plugin — plugins are discovered at runtime via `importlib.metadata.entry_points('rig.devices')`.

## Milestone Summary

**Key Decisions:**

- Entry points as single discovery path (vs code-level registration): eliminates import-time coupling, enables third-party plugins
- Device-level MIDI lifecycle (vs engine-managed): each plugin owns its connection — cleaner separation, no Phase -1 loop
- `MidiConnectionIO` Protocol removed entirely (not just emptied): no longer needed once devices manage their own MIDI
- Three separate packages (rig-chasebliss, rig-hx, rig-morningstar) each own their full device implementation + prompts

**Issues Resolved:**

- Phase -1 MIDI connection loop in `apply.py` — replaced by `Device.setup()`
- Cross-module private symbol imports — eliminated with package extraction
- Dead core code (7 files) — all deleted after plugin migration complete

**Issues Deferred:**

- CI pipeline for independent package publishing (BUILD-03) — future milestone
- Nyquist validation files for v1.1 phases — GSD workflow gap, not code quality issue
- `Device.plan()` / `Device.diff()` stubs still raise NotImplementedError — pre-existing from v1.0

**Technical Debt Incurred:**

- No VERIFICATION.md or SUMMARY.md for Phase 8 plans — GSD workflow gap during execution sprint
