## v1.1 Requirements

### Core

- [x] **CORE-01**: `rig` publishes `rig.devices` entry point group with standard schema — **Satisfied** (Phase 6)
- [x] **CORE-02**: Plugin discovery via `importlib.metadata.entry_points('rig.devices')` at runtime — no hard imports of any plugin — **Satisfied** (Phase 6)
- [x] **CORE-03**: Migration from code-internal `PluginRegistry` to entry-point discovery (backward-compatible path) — **Satisfied** (Phase 6)
- [x] **CORE-04**: `rig` has zero `[project.dependencies]` on any device plugin package — **Satisfied** (Phase 6)

### Device Plugins

- [x] **ANLG-01**: `rig-analog` is a separate pip package with its own `pyproject.toml`, registers AnalogDevice via entry point — **Satisfied** (Phase 7)
- [x] **CHASE-01**: `rig-chasebliss` is a separate pip package with its own `pyproject.toml`, registers ChaseBlissDevice via entry point, manages its own MIDI connection for setup phases — **Satisfied** (Phase 7)
- [x] **MC6-01**: `rig-morningstar` is a separate pip package with its own `pyproject.toml`, registers MC6Device via entry point, manages its own MIDI connection for bank/switch config — **Satisfied** (Phase 7)
- [x] **HX-01**: `rig-hx` is a separate pip package with its own `pyproject.toml`, registers HXStompDevice via entry point — **Satisfied** (Phase 7)
