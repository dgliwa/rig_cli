# Requirements — v1.2 Cleaner Core (ARCHIVED)

**Status:** ✅ COMPLETE — shipped 2026-06-08
**Phases:** 9–13

## Overview

Collapse the multi-file config repo into a single flat `rig.yaml`, strip all plugin-specific code from core, and remove backwards-compat shims introduced in v1.0/v1.1. No backwards compatibility — clean break.

---

## Requirements (All Complete)

### SCHEMA — Single-file rig.yaml

- [x] **SCHEMA-01**: User can define the entire rig in a single `rig.yaml` file (no multi-file layout required) — Phase 10 ✓
- [x] **SCHEMA-02**: Device list order in `rig.yaml` defines the signal chain — no separate signal chain config needed — Phase 10 ✓
- [x] **SCHEMA-03**: `SignalChainPosition` model is removed from core — Phase 9 ✓
- [x] **SCHEMA-04**: A controller device optionally references the device IDs it composes — Phase 10 ✓
- [x] **SCHEMA-05**: Scenes defined inside controller device with `bank`, `switch`, and `presets: {device_id: preset_id}` — Phase 10 ✓
- [x] **SCHEMA-06**: Device identity via `type` only — `manufacturer`/`model` removed from core `Device` — Phase 9 ✓

### MODEL — Core model cleanup

- [x] **MODEL-01**: `ManualConfig`, `MidiConfig`, `ChaseBlissConfig`, `ControllerConfig` removed from core; live only in plugins — Phase 9 ✓
- [x] **MODEL-02**: `Control` and `ControlType` removed from core — Phase 9 ✓
- [x] **MODEL-03**: `Rig` model drops all compat shims: `pedals`, `digital_presets`, `hx_presets`, `analog_presets`, `mc6`, `_controller_device` — Phase 9 ✓
- [x] **MODEL-04**: `Scene` model drops `mc6_bank` and `mc6_switch` fields — Phase 9 ✓

### LOADER — Single-file parser

- [x] **LOADER-01**: `load_rig()` parses a single `rig.yaml` instead of the multi-file directory layout — Phase 10 ✓
- [x] **LOADER-02**: Device construction dispatched to plugins via entry points using the `type` field as the lookup key — Phase 10 ✓

### CLEANUP — Dead code & compat removal

- [x] **CLEANUP-01**: All `TODO: 1.2` markers and their associated dead code removed from the codebase — Phases 11 + 13 ✓
- [x] **CLEANUP-02**: Multi-file config repo support removed — only single `rig.yaml` is accepted — Phase 11 ✓

### DEFER — Deferred item cleanup

- [x] **DEFER-01**: Vestigial `rig generate mc6` CLI command and `rig_morningstar.generator` module removed — Phase 12 ✓
- [x] **DEFER-02**: Unused `composes` validation removed from loader — Phase 12 ✓

### DESIGN — Post-v1.2 design cleanup

- [x] **DESIGN-01**: `Rig.scenes` removed as a stored field — scenes live on controller devices only; `Rig.scenes` is a `@property` that aggregates from controllers — Phase 13 ✓
- [x] **DESIGN-02**: HXStompPreset preset_number lookup unified — `is_hx` branch removed from `compute.py`; `_get_preset_number` handles all preset types uniformly — Phase 13 ✓
