# Milestone v1.4: Architecture & Type Integrity

**Status:** ✅ SHIPPED 2026-06-17
**Phases:** 20-24
**Total Plans:** 5

## Overview

Internal refactor milestone — zero user-visible behavior changes. Eliminates the parallel device model systems, establishes a single typed boundary throughout the plugin/core interface, and closes all test and code-quality debt. Single `Device` Protocol end-to-end; all 4 plugins carry concrete config types and typed preset lists; `appliers/base.py` deleted; 309 tests green.

## Milestone Summary

**Key Decisions:**

- **Phase 20 bundles zero-dependency quick wins** — TYPE-04, TEST-01, QUAL-01, QUAL-02 share no intra-group dependencies; batching reduces phase count without coupling risk
- **TYPE-02 and TYPE-03 in same phase (Phase 21)** — Preset Protocol and concrete plugin configs are co-dependent at the boundary; splitting would leave a half-typed state
- **TYPE-01 (retire legacy Device model) isolated to Phase 22** — most impactful change; depends on Phase 21; isolated blast radius
- **TEST-02 bundled with TYPE-05 in Phase 23** — stdin-capture failures were entangled with dual ApplyContext types; fixing the context first gives correct foundation
- **ActionStatus in models.py, not engine/plugin.py** — plan models are a separate concern from apply-time contracts
- **SkipValidation[list[Preset]] for MC6Device** — Pydantic cannot validate `list[Protocol]` at runtime; `SkipValidation` preserves the typed annotation for static analysis
- **Phase 24 inserted retroactively** — v1.4 audit found 3 gaps (missing Phase 21 VERIFICATION.md, MC6 `list[Any]`, QUAL-01 raw-string assignment sites); Phase 24 closed all three

**Issues Resolved:**

- All 309 tests pass (from 300 at v1.3 close)
- 3 stdin-capture tests that required `-s` flag now pass in normal `make test` run
- MC6Device `list[Any]` preset field eliminated
- Phase 21 VERIFICATION.md written retroactively with live evidence
- `DeviceAction` fully enum-typed — no raw strings at any construction or comparison site

**Issues Deferred:**

- AnalogApplier bypasses `ConfirmationIO` — `prompt_analog()` still calls `input()` directly; analog tests must monkeypatch `builtins.input` (CBA path was fixed in Phase 23; analog parity deferred)

**Technical Debt Incurred:**

- AnalogApplier `ConfirmationIO` gap — not a regression (was broken before); deferred to v1.5
