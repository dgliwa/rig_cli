# Milestone v1.3: Chase Bliss Pedal Support

**Status:** ✅ SHIPPED 2026-06-10
**Phases:** 14-19
**Total Plans:** 6

## Overview

Add full MIDI CC catalog, preset validation, and apply integration for Mood MkII (gap-fill), Wombtone MkII, and Brothers AM Chase Bliss pedals, including a reset-to-defaults step in the apply flow for clean preset creation.

## Milestone Summary

**Key Decisions:**

- CBA-04 + CBA-01 + CBA-02 + CBA-03 grouped in Phase 14 — single phase avoids partial state
- Validation (Phase 15) before reset (Phase 16) — both consume catalog controls; validation precedes any action that touches the device
- All phases live in rig-chasebliss package — no core changes; `Control` model change is additive and backward-compatible
- Imperative validation in apply(), not Pydantic model_validator — catalog lookup at model construction time would couple model layer to plugin
- Phase 17-19 inserted after original 3-phase plan to close gaps identified in audit
- Manufacturer hardcoded as "Chase Bliss Audio" in from_raw_yaml() — model is the only variable discriminator needed
- Auto-populate only when controls list is empty — explicit YAML controls always take precedence

**Issues Resolved:**
- Fixed stale Mood MkII test assertions (Phase 14 catalog expansion left test assertions behind)
- Wired `get_controls()` into production device loading (catalog was defined but had zero callers)
- All 5 phases now have VERIFICATION.md + VALIDATION.md documentation

**Issues Deferred:**
- Validation skips already-saved presets (state-gating by design — avoids redundant re-validation; edited saved YAML params not re-validated until state reset)
- `_send_reset_ccs()` lacks `connected_devices` gate (non-blocking; exceptions caught, asymmetric with `_send_ccs()` gate)

**Technical Debt Incurred:**
- Validation state-gating may miss errors on edited saved presets
- Reset CC asymmetry with `_send_ccs()` gate — no crash risk, but inconsistent
