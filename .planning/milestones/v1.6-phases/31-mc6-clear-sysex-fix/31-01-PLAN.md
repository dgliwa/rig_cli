<objective>
Fix the MC6 clear SysEx default so that `rig apply` writes cleared switch slots to
flash — matching what the MC6 web UI sends — instead of RAM-only. The single root
cause is `save: bool = False` on `clear_preset_messages()` in sysex.py (line 79).
Both sibling functions already default to `save=True`; this change makes the API
consistent and fixes the power-cycle regression.

Purpose: Ensure a cleared switch slot stays cleared after the MC6 is power-cycled.
Output: Corrected sysex.py, updated test assertions, new explicit-save=False tests.
</objective>
