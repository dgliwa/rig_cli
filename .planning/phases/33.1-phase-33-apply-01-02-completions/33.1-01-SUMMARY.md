# Plan 33.1-01 Summary: Apply-01/02 Completions

## What was built

**APPLY-01 — VERIFY branch display**

Added a `console.print` call inside the `ActionStatus.VERIFY` guard in `apply_plan()` (`packages/rig/src/rig/engine/apply.py`). When a device's state already matches the desired preset, the user now sees:

```
  ✓ tumnus: already set to 'edge'
```

No prompt is shown; no `device.apply()` is called.

**APPLY-02 — device_filter cross-scene apply**

Added `device_filter: str | None = None` parameter to `apply_plan()`. When set:
- The inner device-action loop skips any action whose `action.device != device_filter`
- The controller programming phase (`Phase 2`) is gated with `and not device_filter`
- Only the filtered device's state is written back

CLI (`packages/rig/src/rig/cli/commands/apply.py`):
- D-05 guard replaced: `--preset` alone exits 1 with "–preset requires –device"; `--device` alone is now valid
- Added device-only existence check (D-06 variant)
- New routing branch: `elif device:` calls `apply_plan(..., device_filter=device)`
