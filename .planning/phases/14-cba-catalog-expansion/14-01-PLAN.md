<objective>
Expand the rig-chasebliss catalog with complete control definitions for three Chase Bliss Audio
pedals, add a `default` field to the `Control` model, and verify correctness with tests.

Purpose: Preset resets need a factory-default value per control so `apply` can zero out a pedal
before loading a new preset. The `default` field on `Control` is the single source of truth for
that value.

- `catalog.py` — `Control` model gains `default: float | None = None`; Mood MkII catalog is
  corrected (CC102/103 renamed, CC24-29/CC31-33/CC52 added, all existing controls gain defaults);
  `WOMBTONE_MKII_CONTROLS` and `BROTHERS_AM_CONTROLS` are defined and registered in `_CATALOG`.
- `tests/test_catalog.py` — behavioral tests covering model backward-compat, catalog correctness,
  and `get_controls` lookup.
