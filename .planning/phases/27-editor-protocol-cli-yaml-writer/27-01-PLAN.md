<objective>
Phase 27 establishes the editor protocol contract, YAML write-back infrastructure, and the CLI command surface for `rig edit`. The Device Protocol is left unchanged; a companion EditorProtocol is defined alongside it. The YAML writer uses ruamel.yaml round-trip mode to preserve hand-authored comments and field ordering. Analog plugin receives no stub.

Purpose: Deliver the minimal viable edit lifecycle — enter, skeleton interaction, save/discard with atomic write-back — so Phase 28 plugin implementations have a stable protocol and CLI surface to build against.
