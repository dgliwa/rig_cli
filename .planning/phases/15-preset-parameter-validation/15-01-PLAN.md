The function signature:

    def validate_cc_params(
        parameters: dict[str, Any],
        controls: list[Control],
    ) -> list[str]:
        """Validate preset parameters against a device's catalog controls.

        Returns a list of error strings. Empty list means valid.
        """

No new imports are needed — from typing import Any, Literal is already present, and
from rig_chasebliss.catalog import Control is already present.

Validation rules (D-06):

For each (param_name, param_value) in parameters:

1. Unknown name check: Find the control in controls where control.name == param_name
   (case-sensitive). If no control matches, produce:
   f"'{param_name}' is not a valid parameter for this device"

2. Known name — value range check:
   - If control has positions (non-empty list) AND param_value is a str:
     Check if param_value is in control.positions. If not, produce:
     f"'{param_name}': '{param_value}' is not a valid position; allowed: {control.positions}"
   - If control has positions (non-empty list) AND param_value is numeric:
     Validate 0 <= int(param_value) < len(control.positions). If out of range, produce:
     f"'{param_name}': {param_value} is out of range (0-{len(control.positions) - 1}, as position index)"
   - For controls WITHOUT positions (KNOB, DIPSWITCH, SWITCH):
     Coerce param_value to int (same as get_cc_params does) and check
     control.min is not None and int(param_value) < control.min → error, or
     control.max is not None and int(param_value) > control.max → error.
     Error format: f"'{param_name}': {param_value} is out of range ({control.min}-{control.max})"

3. Type coercion safety: Wrap the int() call in a try/except (ValueError, TypeError) for
   the position-index and range checks. If int(param_value) fails:
   f"'{param_name}': '{param_value}' is not a valid numeric value"

Edge cases:
- Empty parameters dict → return [].
- Empty controls list — every param produces "unknown name" error.
- min or max being None — skip that bound check.
