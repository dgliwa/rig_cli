<objective>
Fix two false-positive bugs in the plan/apply pipeline so that state.json-backed device state
eliminates unnecessary prompts and misleading plan output.

**Bug 1 (STATE-01 criterion 2):** `compute.py` assigns `ActionStatus.ANALOG` to analog devices
even when `actual_preset == preset_id`, causing `plan.py` to show a yellow "⚠ set to X (manual)"
warning for a device that does not need to change.

**Bug 2 (STATE-01 criterion 3):** `apply.py` calls `device.apply()` for all actions in a "changed"
scene, including `ActionStatus.VERIFY` actions — prompting the user for devices already at the
correct preset.

Purpose: STATE-01 — `rig plan` must suppress false-positive "changed" signals; `rig apply` must
not re-prompt for already-correct devices.

Output: Three source edits (compute.py, apply.py, plan.py) + five tests (test_plan.py,
test_apply.py, test_cli_plan.py). Delivered as two atomic commits.
</objective>
