### Satisfied Requirements Detail

**CBA-04** — `Control.default: float | None = None` field is backward-compatible (existing call sites get `None`). All catalogs populated with correct defaults (knobs get numeric, switches get `None`). Field is consumed by Phase 15 (as part of controls list) and Phase 16 (`default is not None` filter).

**CBA-01 + CBA-02** — Catalogs completed in Phase 14. Full integration verified:
1. `get_controls()` exported from `catalog.py`
2. `ChaseBlissConfig.model: str | None = None` added (Phase 18)
3. `from_raw_yaml()` calls `get_controls("Chase Bliss Audio", config.model)` when controls empty + model set (Phase 18)
4. Entry point `rig.devices` → `chase_bliss = "rig_chasebliss.device:ChaseBlissDevice"` discovered by `PluginRegistry`
5. Loader calls `model_class.from_raw_yaml(data)` in `_parse_device()`
6. Result: device YAML with `config.model: "Wombtone MkII"` and no explicit controls gets 8 CC controls auto-populated

**CBA-03** — Mood MkII catalog correctly expanded with all missing controls. Stale test assertions (knob count 7→13, CC102/CC103 name swap) fixed in Phase 17. Full test suite: 300 passed, 0 failures.

**CBA-05** — `validate_cc_params()` rejects unknown param names and out-of-range values before any user interaction. Wired into `_detect_cba_setup_for_device()` which gates the apply flow. Runs in dry-run mode (called as part of `setup()` → `_detect_cba_setup_for_device()`). Error format groups by device and preset.

**CBA-06** — `_send_reset_ccs()` sends CC messages for all controls with `c.default is not None and c.midi_cc is not None` immediately before `_send_ccs()`. Per-preset firing (called inside `_build_preset()`). Dry-run displays reset count. Individual CC failures are non-blocking (exceptions caught and logged).

## Technical Debt

### Phase 14 — CBA Catalog Expansion
- **VERIFICATION.md and VALIDATION.md never created** — no formal verification evidence
- **`requirements-completed` frontmatter absent** from 14-01-SUMMARY.md

### Phase 15 — Preset Parameter Validation
- **Validation skips already-saved presets** — `_detect_cba_setup_for_device()` gates on `not ds.presets_saved.get(preset.id)`. If a user edits a saved preset's YAML parameters, the new values are not re-validated until the state entry is cleared. This is by design (state-gating avoids redundant re-application) but means errors in edited presets can slip through.
- **VERIFICATION.md and VALIDATION.md never created**

### Phase 16 — Reset-to-Defaults
- **`_send_reset_ccs()` lacks `connected_devices` gate** — `_send_reset_ccs()` attempts to send CCs without checking `action.device in ctx.connected_devices`, unlike `_send_ccs()` which has this guard. The exceptions are caught (non-blocking, no crash), but the asymmetry means reset CCs are attempted for disconnected devices while preset CCs are not.
- **VERIFICATION.md and VALIDATION.md never created**
